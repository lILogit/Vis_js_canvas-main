Reading the table: A root_cause node may only emit CAUSES, TRIGGERS, or ENABLES edges. An effect node emits only RCDE-planning edges (terminal anchor). A moderator never emits direct causal edges — it modulates strength, not existence.
The RCDE model is not just a layout — it is a constraint layer that determines which node types, archetypes, and edge types are valid in each position.

RRoot cause layer
Types: state, event, concept
Archetypes: root_cause, evidence
Edges out: CAUSES, TRIGGERS, ENABLES
Edges in: none (entry point)
CCausal pathway layer
Types: state, event, blackbox
Archetypes: mechanism, moderator
Edges out: CAUSES, BLOCKS, REDUCES, AMPLIFIES
Edges in: CAUSES, TRIGGERS, ENABLES
DDecision point layer
Types: decision, gate, task, question
Archetypes: question (open), mechanism (task)
Edges out: RESOLVES, DIVERGES_TO, INSTANTIATES
Edges in: CAUSES, PRECONDITION_OF, REQUIRES
EEffect layer
Types: goal, state
Archetypes: effect
Edges out: PRECONDITION_OF (to next chain only)
Edges in: CAUSES, INSTANTIATES, DIVERGES_TO
Cross-layer edge rules
R → C  ✓  allowed (primary flow)
R → D  ✓  allowed (with TRIGGERS only)
R → E  ✗  illegal (must traverse C or D)
C → C  ✓  allowed (chain within pathway)
C → E  ✓  allowed (direct to effect)
D → R  ✗  illegal (no backward arcs)
E → R  ✗  illegal (use temporal snapshot instead)
E → E  ✗  illegal within same chain
Asset & concept nodes are layer-agnostic — they can attach to any layer via REQUIRES or FRAMES. They do not occupy a primary layer slot; they are lateral annotations.
