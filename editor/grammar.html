Rozpočet pro expanzi

Barvy uzlů — míra jistoty

Zelená > 80 % — fakt / prokázané
Modrá 60–80 % — konsensus / pravděpodobné
Oranžová 40–60 % — pracovní hypotéza
Červená < 40 % — spekulace / neznámé
Pravidlo: Barva je vlastnost jistoty, tvar je vlastnost role. Červená QUESTION je nejistá hypotéza. Zelená BLACKBOX je ověřená neznámá. Kombinace jsou záměrné.

Základní vazby (povinné)

ZPŮSOBUJE
šedá / neutrální — hlavní linka
A přímo produkuje B. Základní kauzální hrana. Výchozí volba pro každou vazbu.
UMOŽŇUJE / VYŽADUJE
zelená — podmínka existence
Bez A nemůže B nastat. Neříká, že A přímo způsobí B — jen odstraní bariéru.
BLOKUJE
červená — přerušení —|
A zamezuje B. Intervence TASK nebo DECISION typicky blokuje negativní EVENT.
REALIZUJE
modrá — konstruktivní zásah
TASK nebo DECISION aktivně buduje B. Opak BLOKUJE — opravuje místo brání.

Rozšířené vazby (intenzita a mechanismus)

ZESILUJE / SNIŽUJE
šedá se symbolem ▲ / ▼
A moduluje intenzitu B, ale B by existovalo i bez A. Typické pro feedback smyčky.
SPOUŠTÍ
šedá přerušovaná — trigger
A je podmínkou pro EVENT B. Odlišení od ZPŮSOBUJE: stav spouští událost, ne jiný stav.
PŘEKRAČUJE PRAH
oranžová — threshold crossing
Použij, když kauzální vliv závisí na dosažení konkrétní hodnoty (OOM Kill, kritický stres).
Matice STATE→EVENT: Pokud vidíš STATE → ZPŮSOBUJE → STATE, zeptej se: "Co se změnilo?" — chybí EVENT uzel uprostřed. STATE → SPOUŠTÍ → EVENT → ZPŮSOBUJE → STATE je správná forma.

RCDE workflow — 4 kroky

4
EFFECT — definuj dopad (začni zde)
"Co se stalo nebo co chceme dosáhnout?"
Umísti GOAL nebo výsledný STATE doprava dolů. Pokud dopad ještě nenastal, jde o GOAL (trojúhelník). Pokud nastal, STATE nebo EVENT.
1
ROOT CAUSE — najdi kořen
"Odkud problém pramení? Co to spustilo?"
Umísti doleva nahoru STATE nebo EVENT. Pokud kořen neznáš, vlož QUESTION (hvězda) — je to legitimní uzel, ne chyba. Ptej se "Proč?" max. 5×.
2
CAUSAL PATHWAY — propoj cestu
"Jak kořen vede k dopadu?"
Spoj Root a Effect uzly přes STATE / EVENT. Každý STATE→STATE přechod potřebuje EVENT uprostřed. Max. 4–5 uzlů mezi kořenem a bránou — jinak vlož GATE.
3
DECISION POINT — vlož zásah
"Kde se řetězec dá přerušit nebo přesměrovat?"
Vlož DECISION nebo TASK na místo, kde je vliv na řetězec. TASK + BLOKUJE = prevence. TASK + REALIZUJE = oprava. DECISION vždy větví na ≥ 2 výstupy s různými dopady.

Kontrolní seznam před dokončením

☐ Každý STATE→STATE přechod má EVENT uzel
☐ Každá DECISION má ≥ 2 větvení s efekty
☐ Každá GATE má definovanou podmínku (AND/OR/XOR)
☐ Každý BLACKBOX má barvu jistoty
☐ GOAL uzel uzavírá pravý konec diagramu
☐ QUESTION uzly jsou označeny jako výzkumné úkoly
☐ ASSET uzly mají vazbu UMOŽŇUJE na navazující uzel

Strukturální pravidla

Pravidlo přechodu: stav → událost → stav
Dva STATE uzly nemohou být přímo propojeny přes ZPŮSOBUJE bez mezilehlého EVENT. STATE "Memory Leak" → EVENT "OOM Kill" → STATE "Výpadek" — správně. STATE → STATE — chyba.
Pravidlo hloubky: max 4 uzly bez GATE
Pokud kauzální cesta obsahuje více než 4 uzly bez větvení nebo GATE, vlož GATE s podmínkou. Dlouhé přímé řetězce skrývají latentní větvení.
Pravidlo úplnosti: každá větev má Effect
Každá výstupní větev DECISION nebo GATE musí skončit u STATE, GOAL nebo zpětné smyčky. Slepé větve = neúplné modelování.
Pravidlo BLACKBOX: zapiš symptom, ne příčinu
BLACKBOX obsahuje popis efektu, ne domněnku o příčině. Správně: "Leak nenastal v testu". Špatně: "Pravděpodobně chybný mock".
Pravidlo ASSET: podmínka, ne příčina
ASSET (válec) uzel používej jen tehdy, když jeho absence zastaví řetězec. Spojuj ho vazbou UMOŽŇUJE, nikdy ZPŮSOBUJE.
Pravidlo konzistence barev
Barva uzlu se vztahuje k jistotě jeho obsahu, nikoli k typu uzlu. Pokud STATE "Chronický stres" je jen hypotéza (60 %), je modrý — bez ohledu na tvar.

Temporální orientace

čas →
Root vlevo, Effect vpravo. DECISION a TASK se umísťují na místo v timeline, kde k zásahu reálně dochází — ne vždy na konec. Zpětné smyčky mohou vést zpět doleva.

Časté chyby

✕
State–State bez eventu
"Chronický stres → Kolaps" — chybí EVENT. Správně: "Chronický stres → snižuje → Kvalita spánku → spouští → Kolaps".
✕
Decision bez větvení
DECISION uzel s jednou výstupní hranou není rozhodnutí — je to TASK. Pokud neexistují alespoň dvě volby s různými dopady, změň tvar.
✕
Všechno zelené
Pokud jsou všechny uzly zelené (> 80 %), diagram nedělá epistémii viditelnou. Každý model má nejistá místa — najdi je a obarvi oranžově nebo červeně.
✕
CONCEPT jako náhrada za STATE
"Expanze" je CONCEPT, ale "Přístup k 10 000 novým zákazníkům" je STATE. Pokud jde konkrétizovat, použij STATE. CONCEPT nech pro nezměřitelné abstrakce.
✕
Řetězec bez GATE po 5+ uzlech
Příliš přímý řetězec = přeskočená větvení nebo podmínky. Vlož GATE s otázkou "Nastane vždy?" — pokud ne, existuje podmínka, která patří do modelu.
✕
Smíchání ZPŮSOBUJE a UMOŽŇUJE
ASSET "Rozpočet" → ZPŮSOBUJE → "Expanze" je chyba. Správně: ASSET "Rozpočet" → UMOŽŇUJE → DECISION "Výběr regionu". Bez rozpočtu nelze rozhodovat, ale rozpočet sám expanzi nezpůsobí.
